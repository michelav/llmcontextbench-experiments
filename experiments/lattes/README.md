# Lattes experiments

This directory contains the preserved Lattes baseline used by the SBES 2026 research-track and tools-track papers.

The baseline compares five effective context-provisioning configurations:

- inline HTML (`I-HTML`);
- inline JSON (`I-JSON`);
- local function calling (`Func.`);
- local MCP (`L-MCP`);
- remote MCP (`R-MCP`).

The preserved execution contains 1,200 generated answers and 3,600 individual judge votes. The committed baseline artifacts are the source of truth for every numerical table and figure produced by this directory.

## Reproducibility design

```text
baseline_001/{queries.jsonl,answers.jsonl,judge_votes.jsonl,traces/...}
                                 |
                                 v
                         analyze_lattes.py
                                 |
                                 v
             baseline_001/derived/analysis/*.csv
                                 |
                    +------------+------------+
                    |                         |
                    v                         v
          generate_figures.py           analysis.ipynb
                    |                         |
                    v                         v
 baseline_001/derived/figures/      interactive inspection
```

`analyze_lattes.py` is the single source of truth for derived metrics. It reads the committed baseline artifacts, invokes `extract_observed_calls.py`, and writes normalized CSV files plus a manifest.

`generate_figures.py` reads only those derived CSV files. It does not read raw JSONL files, traces, or expected paper values.

`analysis.ipynb` is an inspection layer over the derived CSV files. It does not recompute the official metrics from raw artifacts.

The previously explored quality/token-cost Pareto figure is intentionally excluded from the official reproducibility workflow.

Expected paper values are verification fixtures only. They are never used as inputs when computing metrics.

## Main research-paper outputs

The relevant quantitative outputs from the research-track paper are:

- **Figure 3a**: latency distribution as a violin plot with an overlaid boxplot;
- **Figure 3b**: question-by-configuration majority-accuracy heatmap;
- **Table 5**: configuration-level effectiveness, cost, latency, and observed calls;
- **Table 6**: model-level effectiveness and execution efficiency by configuration;
- **Table 7**: question-level accuracy, disagreement, calls, tags, and dominant difficulty.

The former file name `tool-paper-table-3a` referred to Table 3(a) of the separate tools-track paper. It is not a table in the research-track article and is no longer generated by this workflow.

## Generated analysis files

Running `analyze_lattes.py` creates:

```text
baseline_001/derived/analysis/
├── lattes_trials.csv
├── lattes_summary_by_configuration.csv
├── lattes_summary_by_model.csv
├── lattes_summary_by_question.csv
├── lattes_summary_by_curriculum.csv
├── lattes_summary_by_tag.csv
├── lattes_summary_by_judge.csv
├── table-5.csv
├── table-6.csv
├── table-7.csv
└── lattes_analysis_manifest.json
```

The three `table-*.csv` files are formatted to correspond directly to Tables 5, 6, and 7 of the research paper. Their numerical values are computed from the preserved baseline. The qualitative `Dominant difficulty` column in Table 7 records the interpretations reported in the article.

Observed calls are extracted into:

```text
baseline_001/derived/calls/
├── observed_tool_calls_by_run.csv
├── observed_tool_calls_long.csv
└── observed_tool_calls_summary.csv
```

The analysis script invokes the extractor automatically. Running the extractor separately is useful only for inspecting call-level evidence.

## Requirements

Offline analysis requires:

- Python 3.11 or 3.12;
- pandas;
- NumPy;
- Matplotlib;
- `just`, as the repository-level command runner;
- Jupyter, only for the notebook.

Example Python environment:

```bash
python -m venv .venv
source .venv/bin/activate
pip install pandas numpy matplotlib jupyter
```

Check the available recipes with:

```bash
just --version
just --list
```

`just` is only an orchestration layer. Every recipe delegates to the documented Python scripts, which can also be executed directly.

Offline processing does not call model providers or the remote MCP server.

## Reproduce Tables 5–7 and Figure 3

Run from the repository root:

```bash
just lattes-all
```

The workflow can also be run in separate stages:

```bash
just lattes-analyze
just lattes-figures
```

Equivalent direct commands:

```bash
python experiments/lattes/analyze_lattes.py \
  experiments/lattes/baseline_001 \
  --output-dir experiments/lattes/baseline_001/derived/analysis

python experiments/lattes/generate_figures.py \
  experiments/lattes/baseline_001/derived/analysis \
  --output-dir experiments/lattes/baseline_001/derived/figures
```

To generate selected figures:

```bash
python experiments/lattes/generate_figures.py \
  experiments/lattes/baseline_001/derived/analysis \
  --output-dir experiments/lattes/baseline_001/derived/figures \
  --only latency heatmap
```

Valid values for `--only` are:

```text
latency heatmap combined
```

## Inspect the derived results in Jupyter

Generate the CSVs first, then open the notebook:

```bash
just lattes-analyze
jupyter lab experiments/lattes/analysis.ipynb
```

For non-interactive execution:

```bash
just lattes-notebook
```

The notebook displays Tables 5, 6, and 7 directly from the generated CSV files. It fails explicitly when required derived files are absent and does not silently fall back to raw baseline artifacts.

## Verify Table 5 against the article fixture

`expected/table-5-lattes.csv` is read only after baseline-derived analysis has completed:

```bash
just lattes-verify
```

Equivalent direct command:

```bash
python experiments/lattes/analyze_lattes.py \
  experiments/lattes/baseline_001 \
  --output-dir experiments/lattes/baseline_001/derived/analysis \
  --expected expected/table-5-lattes.csv \
  --verify
```

A mismatch causes a non-zero exit status. The expected CSV never replaces or completes missing baseline data.

## Inspect observed operation calls

The analysis script performs this extraction automatically. To run it independently:

```bash
python experiments/lattes/extract_observed_calls.py \
  experiments/lattes/baseline_001 \
  --output-dir experiments/lattes/baseline_001/derived/calls
```

For an additional recursive scan of raw provider responses:

```bash
python experiments/lattes/extract_observed_calls.py \
  experiments/lattes/baseline_001 \
  --output-dir experiments/lattes/baseline_001/derived/calls \
  --scan-raw-response
```

## Re-running the experiment

The preserved artifacts are sufficient to reproduce the reported offline analyses. Re-running the complete experiment is optional and requires provider credentials, a compatible remote MCP server, network access, sufficient rate limits and budget, and the archived software and dataset snapshots.

Do not substitute the current external tool or dataset state when claiming exact reproduction of this baseline.
