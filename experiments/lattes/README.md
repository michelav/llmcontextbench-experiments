# Lattes experiments

This directory contains the preserved Lattes baseline used by the SBES 2026 research-track and tools-track papers.

The baseline compares five effective context-provisioning configurations:

- inline HTML (`I-HTML`);
- inline JSON (`I-JSON`);
- local function calling (`Func.`);
- local MCP (`L-MCP`);
- remote MCP (`R-MCP`).

The preserved execution contains 1,200 generated answers and 3,600 individual judge votes. The committed baseline artifacts are the source of truth for every table and figure produced by this directory.

## Reproducibility design

The official processing path is deliberately split into two stages:

```text
baseline_001/{answers.jsonl,judge_votes.jsonl,traces/...}
                         |
                         v
                 analyze_lattes.py
                         |
                         v
baseline_001/derived/analysis/*.csv
                         |
                         +----------------------+
                         |                      |
                         v                      v
              generate_figures.py        analysis.ipynb
                         |                      |
                         v                      v
baseline_001/derived/figures/       interactive inspection
```

`analyze_lattes.py` is the single source of truth for derived metrics. It reads the committed baseline artifacts, invokes `extract_observed_calls.py`, and writes normalized CSV files plus a manifest.

`generate_figures.py` reads only those derived CSV files. It does not read raw JSONL files, traces, or expected paper values.

`analysis.ipynb` is an inspection layer over the derived CSV files. It does not recompute the official metrics from the raw artifacts.

The previously explored quality/token-cost Pareto figure is intentionally excluded from the reproducibility workflow and is not generated by the scripts or notebook.

Expected paper values are verification fixtures only. They are never used as inputs when computing metrics.

## Requirements

Offline analysis requires:

- Python 3.11 or 3.12;
- pandas;
- NumPy;
- Matplotlib;
- `just`, as the repository-level command runner;
- Jupyter, only for the notebook.

Example Python environment:

```bash
python -m venv .venv
source .venv/bin/activate
pip install pandas numpy matplotlib jupyter
```

Install `just` using the package manager for your operating system, then confirm that the repository recipes are available:

```bash
just --version
just --list
```

`just` is only an orchestration layer: every recipe delegates to the documented Python scripts. The scripts can still be executed directly when `just` is unavailable.

Offline processing does not call model providers or the remote MCP server.

## Reproduce the analysis and figures

Run from the repository root:

```bash
just lattes-all
```

The workflow can also be run in separate stages:

```bash
just lattes-analyze
just lattes-figures
```

Equivalent direct commands:

```bash
python experiments/lattes/analyze_lattes.py \
  experiments/lattes/baseline_001 \
  --output-dir experiments/lattes/baseline_001/derived/analysis

python experiments/lattes/generate_figures.py \
  experiments/lattes/baseline_001/derived/analysis \
  --output-dir experiments/lattes/baseline_001/derived/figures
```

The figure generator currently produces:

- question-by-strategy majority-accuracy heatmap;
- latency distribution by strategy;
- question-level difficulty and judge-disagreement chart.

To generate selected figures directly:

```bash
python experiments/lattes/generate_figures.py \
  experiments/lattes/baseline_001/derived/analysis \
  --output-dir experiments/lattes/baseline_001/derived/figures \
  --only heatmap latency
```

Valid values for `--only` are:

```text
heatmap latency disagreement
```

## Inspect the derived results in Jupyter

Generate the CSVs first, then open the notebook:

```bash
just lattes-analyze
jupyter lab experiments/lattes/analysis.ipynb
```

For non-interactive execution:

```bash
just lattes-notebook
```

The notebook fails explicitly when required derived files are absent. It does not silently fall back to the raw baseline artifacts.

## Verify against the published table

Expected files are optional verification fixtures. They are read only after the baseline-derived analysis has completed.

```bash
just lattes-verify
```

Equivalent direct command:

```bash
python experiments/lattes/analyze_lattes.py \
  experiments/lattes/baseline_001 \
  --output-dir experiments/lattes/baseline_001/derived/analysis \
  --expected expected/table-3a-lattes.csv \
  --verify
```

A mismatch causes a non-zero exit status when `--verify` is supplied. The expected CSV never replaces or completes missing baseline data.

## Inspect observed operation calls

The analysis script performs this extraction automatically. To run it independently:

```bash
python experiments/lattes/extract_observed_calls.py \
  experiments/lattes/baseline_001 \
  --output-dir experiments/lattes/baseline_001/derived/calls
```

For an additional recursive scan of raw provider responses:

```bash
python experiments/lattes/extract_observed_calls.py \
  experiments/lattes/baseline_001 \
  --output-dir experiments/lattes/baseline_001/derived/calls \
  --scan-raw-response
```

## Re-running the experiment

The preserved artifacts are sufficient to reproduce the reported offline analyses. Re-running the complete experiment is optional and requires:

- provider credentials for the configured OpenAI, Google, and Anthropic models;
- a compatible remote MCP server;
- network access, provider rate limits, and budget;
- the archived tool and dataset snapshots;
- acceptance that SaaS model updates may change newly generated outputs.

Do not substitute the current external tool or dataset state when claiming exact reproduction of this baseline.
